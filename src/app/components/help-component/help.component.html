Help Technical Info 

The site is composed of two components:
- an API
- a frontEnd

Both are available on github : 
genealogy-api
genealogy-web

Feel free to contribute.




STAGING Env:
When a commit is done on staging environnment,
a build is triggered using heroku and binaries are deployed to :
genealogy-api-staging
genealogy-web-staging

PROD:
When a commit is done on prod environnment,
a build is triggered using travis-ci 

travis - genealogy-api
travis - genealogy-web

Once tests executed successfully, it is deployed to 
 Res01.com
Api


Note both services are hosted for free on Heroku.
After 30 mins of inactivity, the server is sleeping, and takes a while to restart.
I may switch to paid hosting if needed.
Personal Data 
    By default, names and year of birth/death are visible to everyone.
    Other informations are only visible logged users.

    To register, you should find your profile first (if it exists) and then
    claim the profile.

    Roles will be implemented later to avoid sharing all the informations with
    everyone.
